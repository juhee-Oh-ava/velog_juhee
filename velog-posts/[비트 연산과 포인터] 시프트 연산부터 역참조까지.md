<h1 id="오늘-배운-내용-정리">오늘 배운 내용 정리</h1>
<h2 id="부호-비트">부호 비트</h2>
<p>부호 비트는 부호 있는 자료형, 즉 <code>signed</code> 자료형에서 가장 왼쪽에 있는 첫 번째 비트이다.</p>
<ul>
<li>부호 비트가 <code>1</code>이면 음수</li>
<li>부호 비트가 <code>0</code>이면 양수</li>
</ul>
<p>예를 들어 <code>signed char</code>가 8비트라고 하면 다음과 같다.</p>
<pre><code class="language-c">1000 0011</code></pre>
<p>이 값은 <code>signed char</code>에서 <code>-125</code>를 의미한다.</p>
<hr />
<h2 id="signed-char-1000-0011이--125인-이유">signed char 1000 0011이 -125인 이유</h2>
<p><code>signed char</code>는 보통 8비트이고, 표현 범위는 다음과 같다.</p>
<pre><code class="language-text">-128 ~ 127</code></pre>
<p><code>1000 0011</code>은 맨 앞 비트가 <code>1</code>이므로 음수이다.</p>
<p>컴퓨터는 보통 음수를 표현할 때 <code>2의 보수</code> 방식을 사용한다.</p>
<p><code>1000 0011</code>이 어떤 음수인지 구하려면 다음처럼 계산할 수 있다.</p>
<pre><code class="language-text">1000 0011

1. 비트를 반전한다.
0111 1100

2. 1을 더한다.
0111 1101

3. 0111 1101은 125이다.

따라서 원래 값은 -125이다.</code></pre>
<p>즉,</p>
<pre><code class="language-text">1000 0011 = -125</code></pre>
<hr />
<h2 id="오른쪽-시프트">오른쪽 시프트</h2>
<p>오른쪽 시프트는 비트를 오른쪽으로 이동시키는 연산이다.</p>
<pre><code class="language-c">&gt;&gt;</code></pre>
<p>예를 들어 다음과 같은 값이 있다고 하자.</p>
<pre><code class="language-text">1000 0011</code></pre>
<p>이 값을 오른쪽으로 5번 이동하면 다음과 같이 비트들이 오른쪽으로 밀린다.</p>
<pre><code class="language-text">1000 0011 &gt;&gt; 5</code></pre>
<p>부호 있는 자료형의 음수는 오른쪽으로 시프트할 때, 왼쪽의 빈 공간이 부호 비트 값으로 채워지는 경우가 많다.</p>
<p><code>1000 0011</code>의 부호 비트는 <code>1</code>이므로 빈 공간이 <code>1</code>로 채워진다.</p>
<pre><code class="language-text">1000 0011 &gt;&gt; 5

1111 1100</code></pre>
<p>이처럼 음수의 오른쪽 시프트는 왼쪽 빈 공간이 <code>1</code>로 채워져 음수 상태를 유지하는 경우가 많다.</p>
<blockquote>
<p>단, C 언어에서 <code>signed</code> 음수의 오른쪽 시프트 결과는 컴파일러 구현에 따라 달라질 수 있다.<br />대부분의 컴파일러는 부호 비트를 채우는 방식인 산술 시프트를 사용한다.</p>
</blockquote>
<hr />
<h2 id="signed와-unsigned의-오른쪽-시프트-차이">signed와 unsigned의 오른쪽 시프트 차이</h2>
<h3 id="signed-양수의-오른쪽-시프트">signed 양수의 오른쪽 시프트</h3>
<p>부호 있는 자료형에서 부호 비트가 <code>0</code>인 양수는 오른쪽으로 이동하면 왼쪽 빈 공간이 <code>0</code>으로 채워진다.</p>
<pre><code class="language-text">0111 0001 &gt;&gt; 2

0001 1100</code></pre>
<p><code>0111 0001</code>은 10진수로 <code>113</code>이다.</p>
<p>오른쪽으로 2번 이동하면 대략 4로 나눈 것과 비슷한 효과가 난다.</p>
<pre><code class="language-text">113 &gt;&gt; 2 = 28</code></pre>
<hr />
<h3 id="unsigned의-오른쪽-시프트">unsigned의 오른쪽 시프트</h3>
<p><code>unsigned</code> 자료형은 부호 비트가 없다.</p>
<p>따라서 오른쪽으로 시프트하면 왼쪽의 빈 공간은 항상 <code>0</code>으로 채워진다.</p>
<pre><code class="language-c">unsigned char n = 0b10000011;</code></pre>
<pre><code class="language-text">1000 0011 &gt;&gt; 5

0000 0100</code></pre>
<p><code>signed</code>에서는 맨 앞의 <code>1</code>을 부호 비트로 해석할 수 있지만, <code>unsigned</code>에서는 그냥 값의 일부로 본다.</p>
<hr />
<h2 id="왼쪽-시프트와-오버플로우">왼쪽 시프트와 오버플로우</h2>
<p>왼쪽 시프트는 비트를 왼쪽으로 이동시키는 연산이다.</p>
<pre><code class="language-c">&lt;&lt;</code></pre>
<p>예를 들어 <code>signed char</code>에서 다음 값은 양수 <code>113</code>이다.</p>
<pre><code class="language-text">0111 0001</code></pre>
<p>이 값을 왼쪽으로 2번 이동하면 다음과 같다.</p>
<pre><code class="language-text">0111 0001 &lt;&lt; 2

1100 0100</code></pre>
<p>왼쪽으로 이동하면서 <code>1</code>이 부호 비트 자리를 차지했다.</p>
<p>그래서 부호 있는 자료형에서는 음수처럼 해석될 수 있다.</p>
<p>이런 상황은 오버플로우와 관련이 있다.</p>
<blockquote>
<p>주의<br />C 언어에서 <code>signed</code> 정수의 왼쪽 시프트 결과가 표현 범위를 벗어나면 정의되지 않은 동작이 될 수 있다.<br />즉, 항상 “양수가 음수로 바뀐다”라고 단정하기보다는 위험한 코드라고 이해하는 것이 좋다.</p>
</blockquote>
<hr />
<h2 id="오버플로우-예시">오버플로우 예시</h2>
<p>오버플로우는 자료형이 표현할 수 있는 범위를 넘어섰을 때 발생한다.</p>
<h3 id="예시-1-signed-char의-최댓값-넘기기">예시 1. signed char의 최댓값 넘기기</h3>
<p><code>signed char</code>의 범위가 보통 <code>-128 ~ 127</code>이라고 하자.</p>
<pre><code class="language-c">#include &lt;stdio.h&gt;

int main(void)
{
    signed char n = 127;

    n = n + 1;

    printf(&quot;%d\n&quot;, n);

    return 0;
}</code></pre>
<p><code>127</code>은 <code>signed char</code>가 표현할 수 있는 가장 큰 값이다.</p>
<p>여기서 <code>1</code>을 더하면 <code>128</code>이 되어야 하지만, <code>signed char</code>는 <code>128</code>을 표현할 수 없다.</p>
<p>많은 환경에서는 다음처럼 보일 수 있다.</p>
<pre><code class="language-text">0111 1111   // 127
+0000 0001
-----------
1000 0000   // signed char에서는 보통 -128처럼 해석됨</code></pre>
<p>하지만 C 언어에서 signed 정수 오버플로우는 정의되지 않은 동작이다.</p>
<hr />
<h3 id="예시-2-signed-char의-최솟값보다-작아지기">예시 2. signed char의 최솟값보다 작아지기</h3>
<pre><code class="language-c">#include &lt;stdio.h&gt;

int main(void)
{
    signed char n = -128;

    n = n - 1;

    printf(&quot;%d\n&quot;, n);

    return 0;
}</code></pre>
<p><code>signed char</code>의 최솟값이 <code>-128</code>일 때, 여기서 <code>1</code>을 더 빼면 <code>-129</code>가 된다.</p>
<p>하지만 <code>signed char</code>는 <code>-129</code>를 표현할 수 없기 때문에 오버플로우가 발생한다.</p>
<hr />
<h3 id="예시-3-int-범위를-넘는-계산">예시 3. int 범위를 넘는 계산</h3>
<pre><code class="language-c">#include &lt;stdio.h&gt;

int main(void)
{
    int n = 1000000000;

    n = n * 3;

    printf(&quot;%d\n&quot;, n);

    return 0;
}</code></pre>
<p><code>int</code>가 표현할 수 있는 범위를 넘으면 오버플로우가 발생한다.</p>
<hr />
<h2 id="음수를-왼쪽-시프트하는-경우">음수를 왼쪽 시프트하는 경우</h2>
<p>처음에는 “이미 부호 비트가 1인 음수는 왼쪽으로 이동해도 계속 음수이다”라고 생각할 수 있다.</p>
<p>하지만 항상 그렇지는 않다.</p>
<p>예를 들어 다음 값을 보자.</p>
<pre><code class="language-text">1000 0011</code></pre>
<p>이 값을 왼쪽으로 2번 이동하면 다음처럼 될 수 있다.</p>
<pre><code class="language-text">1000 0011 &lt;&lt; 2

0000 1100</code></pre>
<p>왼쪽으로 밀려나면서 기존의 부호 비트 <code>1</code>이 사라지고, 새 부호 비트가 <code>0</code>이 될 수도 있다.</p>
<p>그러면 결과가 양수처럼 보일 수도 있다.</p>
<p>따라서 부호 있는 자료형을 시프트할 때는 다음을 조심해야 한다.</p>
<ul>
<li>양수가 음수처럼 보일 수 있다.</li>
<li>음수가 양수처럼 보일 수 있다.</li>
<li>표현 범위를 넘으면 오버플로우가 발생할 수 있다.</li>
<li>C 언어에서는 signed 정수의 위험한 시프트가 정의되지 않은 동작이 될 수 있다.</li>
</ul>
<hr />
<h1 id="do-while문">do while문</h1>
<p><code>do while</code>문은 조건식의 결과와 상관없이 <code>do</code> 부분의 코드를 무조건 한 번 실행한다.</p>
<p>그 다음 조건식을 검사한다.</p>
<p>조건식이 참이면 반복하고, 거짓이면 반복문을 끝낸 뒤 다음 코드를 실행한다.</p>
<pre><code class="language-c">do
{
    실행할 코드
    변화식
} while (조건식);</code></pre>
<hr />
<h2 id="do-while-예시">do while 예시</h2>
<pre><code class="language-c">#include &lt;stdio.h&gt;

int main(void)
{
    int i = 10;

    do
    {
        printf(&quot;i의 값: %d\n&quot;, i);
        i++;
    } while (i &lt; 5);

    return 0;
}</code></pre>
<p><code>i</code>의 초기값은 <code>10</code>이다.</p>
<p>조건식 <code>i &lt; 5</code>는 처음부터 거짓이다.</p>
<p>하지만 <code>do while</code>문은 조건식을 검사하기 전에 먼저 실행되므로 출력이 한 번은 발생한다.</p>
<pre><code class="language-text">i의 값: 10</code></pre>
<p>즉, <code>do while</code>문은 조건식이 처음부터 거짓이어도 최소 한 번은 실행된다.</p>
<hr />
<h1 id="break">break</h1>
<p><code>break</code>는 반복문이나 <code>switch</code>문에서 제어 흐름을 벗어나기 위해 사용한다.</p>
<p><code>break</code>를 사용할 수 있는 곳은 다음과 같다.</p>
<ul>
<li><code>for</code></li>
<li><code>while</code></li>
<li><code>do while</code></li>
<li><code>switch</code></li>
</ul>
<pre><code class="language-c">#include &lt;stdio.h&gt;

int main(void)
{
    for (int i = 1; i &lt;= 10; i++)
    {
        if (i == 5)
        {
            break;
        }

        printf(&quot;%d\n&quot;, i);
    }

    return 0;
}</code></pre>
<p>출력 결과는 다음과 같다.</p>
<pre><code class="language-text">1
2
3
4</code></pre>
<p><code>i</code>가 <code>5</code>가 되는 순간 <code>break</code>가 실행되어 반복문을 빠져나간다.</p>
<hr />
<h1 id="continue">continue</h1>
<p><code>continue</code>는 반복문 안에서 사용한다.</p>
<p><code>continue</code>를 만나면 <code>continue</code> 아래의 코드는 실행하지 않고, 바로 다음 반복으로 넘어간다.</p>
<pre><code class="language-c">#include &lt;stdio.h&gt;

int main(void)
{
    for (int i = 1; i &lt;= 5; i++)
    {
        if (i == 3)
        {
            continue;
        }

        printf(&quot;%d\n&quot;, i);
    }

    return 0;
}</code></pre>
<p>출력 결과는 다음과 같다.</p>
<pre><code class="language-text">1
2
4
5</code></pre>
<p><code>i</code>가 <code>3</code>일 때는 <code>continue</code> 아래에 있는 <code>printf</code>가 실행되지 않는다.</p>

<h1 id="포인터">포인터</h1>
<p>변수는 컴퓨터의 메모리에 생성된다.</p>
<p>변수의 메모리 주소를 구할 때는 변수 앞에 <code>&amp;</code>를 붙인다.</p>
<pre><code class="language-c">&amp;변수이름</code></pre>
<p>메모리 주소는 보통 16진수 형태로 출력된다.</p>
<p>또한 메모리 주소는 고정된 값이 아니다.<br />프로그램을 실행할 때마다 달라질 수 있다.</p>
<p>C 언어에서 메모리 주소는 포인터 변수에 저장한다.</p>
<hr />
<h2 id="포인터-선언">포인터 선언</h2>
<p>포인터 변수를 선언할 때는 자료형 뒤에 <code>*</code>를 붙인다.</p>
<pre><code class="language-c">int *ptr;
int* ptr;
int * ptr;</code></pre>
<p>위 세 가지는 모두 같은 의미이다.</p>
<p>하지만 보통은 다음 형태를 많이 사용한다.</p>
<pre><code class="language-c">int *ptr;</code></pre>
<p>여기서 의미는 다음과 같다.</p>
<ul>
<li><code>ptr</code>은 포인터 변수이다.</li>
<li><code>ptr</code>은 <code>int</code>형 변수의 주소를 저장할 수 있다.</li>
<li><code>ptr</code>이 가리키는 곳에 있는 값은 <code>int</code>로 해석된다.</li>
</ul>
<blockquote>
<p>참고<br /><code>int *ptr</code>에서 <code>*</code>는 “ptr이 포인터 변수다”라는 뜻으로 이해하면 된다.</p>
</blockquote>
<hr />
<h2 id="포인터-코드-예시">포인터 코드 예시</h2>
<pre><code class="language-c">#include &lt;stdio.h&gt;

int main(void)
{
    int num = 10;       // int형 변수 num을 선언하고 10 저장
    int *ptr = &amp;num;    // num의 주소를 ptr에 저장

    printf(&quot;num의 값: %d\n&quot;, num);
    printf(&quot;num의 주소: %p\n&quot;, (void *)&amp;num);
    printf(&quot;ptr에 저장된 주소: %p\n&quot;, (void *)ptr);
    printf(&quot;ptr이 가리키는 값: %d\n&quot;, *ptr);

    return 0;
}</code></pre>
<p>예상 출력 형태는 다음과 비슷하다.</p>
<pre><code class="language-text">num의 값: 10
num의 주소: 0x16fdff32c
ptr에 저장된 주소: 0x16fdff32c
ptr이 가리키는 값: 10</code></pre>
<p><code>ptr</code>에는 <code>num</code>의 주소가 저장되어 있다.</p>
<p>따라서 <code>ptr</code>과 <code>&amp;num</code>은 같은 주소를 출력한다.</p>
<p>그리고 <code>*ptr</code>은 <code>ptr</code>이 가리키는 주소에 접근해서 그 안에 저장된 값인 <code>10</code>을 가져온다.</p>
<hr />
<h2 id="포인터와-변수의-관계-그림">포인터와 변수의 관계 그림</h2>
<pre><code class="language-c">int num = 10;
int *ptr = &amp;num;</code></pre>
<p>위 코드는 다음과 같은 관계를 만든다.</p>
<pre><code class="language-text">ptr 변수
+----------------+
| 0x1000         |
+----------------+
        |
        | ptr은 num의 주소를 저장한다.
        v

주소 0x1000
+----------------+
| num = 10       |
+----------------+</code></pre>
<p>정리하면 다음과 같다.</p>
<pre><code class="language-text">num   : 실제 값 10을 저장하는 변수
&amp;num  : num이 저장된 메모리 주소
ptr   : num의 주소를 저장하는 포인터 변수
*ptr  : ptr이 가리키는 곳에 있는 값</code></pre>
<hr />
<h1 id="역참조">역참조</h1>
<p>포인터는 변수의 주소를 저장한다.</p>
<p>역참조는 포인터에 저장된 주소로 직접 접근해서 값을 가져오거나 수정하는 것이다.</p>
<p>역참조 연산자는 <code>*</code>이다.</p>
<pre><code class="language-c">*ptr</code></pre>
<p><code>ptr</code>이 주소를 저장하고 있다면, <code>*ptr</code>은 그 주소에 있는 실제 값에 접근한다.</p>
<hr />
<h2 id="역참조-코드-예시">역참조 코드 예시</h2>
<pre><code class="language-c">#include &lt;stdio.h&gt;

int main(void)
{
    int num = 10;       // int형 변수 num에 10 저장
    int *ptr = &amp;num;    // ptr은 num의 주소를 저장

    printf(&quot;수정 전 num: %d\n&quot;, num);

    *ptr = 20;          // ptr이 가리키는 공간, 즉 num에 20 저장

    printf(&quot;수정 후 num: %d\n&quot;, num);

    return 0;
}</code></pre>
<p>출력 결과는 다음과 같다.</p>
<pre><code class="language-text">수정 전 num: 10
수정 후 num: 20</code></pre>
<p><code>*ptr = 20;</code>은 <code>ptr</code>이 가리키는 메모리 공간에 <code>20</code>을 저장한다.</p>
<p>즉, 실제로는 <code>num</code>의 값이 바뀐다.</p>
<hr />
<h2 id="역참조-그림">역참조 그림</h2>
<pre><code class="language-c">int num = 10;
int *ptr = &amp;num;</code></pre>
<p>처음 상태는 다음과 같다.</p>
<pre><code class="language-text">ptr
 |
 v
+----------------+
| num = 10       |
+----------------+</code></pre>
<p>여기서 다음 코드를 실행한다.</p>
<pre><code class="language-c">*ptr = 20;</code></pre>
<p>그러면 <code>ptr</code>이 가리키는 공간의 값이 바뀐다.</p>
<pre><code class="language-text">ptr
 |
 v
+----------------+
| num = 20       |
+----------------+</code></pre>
<p><code>ptr</code> 자체가 바뀐 것이 아니라, <code>ptr</code>이 가리키는 곳의 값이 바뀐 것이다.</p>
<hr />
<h1 id="포인터-타입과-역참조-타입">포인터 타입과 역참조 타입</h1>
<p>다음 코드를 보자.</p>
<pre><code class="language-c">int num = 10;
int *ptr = &amp;num;</code></pre>
<p><code>ptr</code>의 자료형은 다음과 같다.</p>
<pre><code class="language-c">int *</code></pre>
<p>즉, <code>ptr</code>은 <code>int</code>를 가리키는 포인터이다.</p>
<p>그런데 <code>ptr</code>을 역참조하면 다음과 같다.</p>
<pre><code class="language-c">*ptr</code></pre>
<p><code>*ptr</code>의 자료형은 <code>int</code>이다.</p>
<p>즉, <code>int *</code>에서 역참조를 하면 실제 값의 자료형인 <code>int</code>가 된다.</p>
<pre><code class="language-text">ptr   : int *
*ptr  : int</code></pre>
<p>그래서 역참조 연산자는 <code>pointer to int</code>에서 <code>pointer to</code>를 제거하고 본래 자료형인 <code>int</code>로 접근하게 해 준다고 이해할 수 있다.</p>
<hr />
<h2 id="역참조-연산자가-자료형을-바꾸는-예시">역참조 연산자가 자료형을 바꾸는 예시</h2>
<pre><code class="language-c">#include &lt;stdio.h&gt;

int main(void)
{
    int num = 10;
    int *ptr = &amp;num;

    printf(&quot;sizeof(ptr): %zu\n&quot;, sizeof(ptr));
    printf(&quot;sizeof(*ptr): %zu\n&quot;, sizeof(*ptr));

    return 0;
}</code></pre>
<p>예상 출력은 환경에 따라 다르지만, 64비트 환경에서는 보통 다음과 비슷하다.</p>
<pre><code class="language-text">sizeof(ptr): 8
sizeof(*ptr): 4</code></pre>
<p>이유는 다음과 같다.</p>
<pre><code class="language-text">ptr은 int 포인터이므로 주소 크기만큼의 크기를 가진다.
64비트 환경에서는 보통 포인터 크기가 8바이트이다.

*ptr은 int 값이므로 int 크기만큼의 크기를 가진다.
대부분의 환경에서 int는 4바이트이다.</code></pre>
<p>즉, <code>ptr</code>과 <code>*ptr</code>은 자료형이 다르다.</p>
<pre><code class="language-text">ptr   : int *
*ptr  : int</code></pre>
<hr />
<h2 id="역참조로-값-가져오기와-저장하기">역참조로 값 가져오기와 저장하기</h2>
<p>역참조 연산자는 값을 가져올 수도 있고, 값을 저장할 수도 있다.</p>
<pre><code class="language-c">#include &lt;stdio.h&gt;

int main(void)
{
    int num = 10;
    int *ptr = &amp;num;

    int a = *ptr;   // ptr이 가리키는 값을 가져와서 a에 저장
    *ptr = 30;      // ptr이 가리키는 공간에 30 저장

    printf(&quot;a: %d\n&quot;, a);
    printf(&quot;num: %d\n&quot;, num);

    return 0;
}</code></pre>
<p>출력 결과는 다음과 같다.</p>
<pre><code class="language-text">a: 10
num: 30</code></pre>
<p><code>a = *ptr;</code>은 값을 가져오는 코드이다.</p>
<p><code>*ptr = 30;</code>은 값을 저장하는 코드이다.</p>
<hr />
<h1 id="변수의-자료형과-포인터를-역참조한-자료형이-다르면">변수의 자료형과 포인터를 역참조한 자료형이 다르면?</h1>
<p>변수의 자료형과 포인터를 역참조한 자료형은 맞아야 한다.</p>
<p>예를 들어 <code>int</code> 변수의 주소는 <code>int *</code> 포인터에 저장해야 한다.</p>
<pre><code class="language-c">int num = 10;
int *ptr = &amp;num;</code></pre>
<p>이것은 올바른 코드이다.</p>
<p>하지만 다음처럼 자료형이 다르면 문제가 생긴다.</p>
<pre><code class="language-c">int num = 10;
double *ptr = &amp;num;   // 잘못된 코드</code></pre>
<p><code>num</code>은 <code>int</code>형 변수이다.</p>
<p>그런데 <code>ptr</code>은 <code>double</code>을 가리키는 포인터이다.</p>
<p>즉, <code>ptr</code>은 해당 주소에 있는 값을 <code>double</code>로 해석하려고 한다.</p>
<p>하지만 실제 메모리에는 <code>int</code> 값이 들어 있다.</p>
<p>이런 식으로 자료형이 맞지 않으면 다음과 같은 문제가 생길 수 있다.</p>
<ul>
<li>컴파일 경고가 발생할 수 있다.</li>
<li>잘못된 값이 출력될 수 있다.</li>
<li>메모리를 잘못 읽거나 쓸 수 있다.</li>
<li>프로그램이 비정상 종료될 수 있다.</li>
<li>정의되지 않은 동작이 발생할 수 있다.</li>
</ul>
<hr />
<h2 id="잘못된-포인터-타입-예시">잘못된 포인터 타입 예시</h2>
<pre><code class="language-c">#include &lt;stdio.h&gt;

int main(void)
{
    int num = 10;

    double *ptr = (double *)&amp;num;   // 억지로 double 포인터로 변환

    printf(&quot;%f\n&quot;, *ptr);           // int 메모리를 double처럼 해석함

    return 0;
}</code></pre>
<p>이 코드는 위험하다.</p>
<p><code>num</code>은 <code>int</code>라서 보통 4바이트를 사용한다.</p>
<p>하지만 <code>double</code>은 보통 8바이트를 사용한다.</p>
<p>따라서 <code>*ptr</code>을 하면 컴퓨터는 <code>num</code>이 있는 위치부터 8바이트를 읽으려고 한다.</p>
<pre><code class="language-text">실제 메모리

int num = 10;

+------+------+------+------+
| int 값 10           |
+------+------+------+------+
  4바이트</code></pre>
<p>그런데 <code>double *</code>로 해석하면 다음처럼 읽으려고 한다.</p>
<pre><code class="language-text">+------+------+------+------+------+------+------+------+
| double 값이라고 생각하고 8바이트를 읽으려고 함        |
+------+------+------+------+------+------+------+------+</code></pre>
<p>실제 자료형과 다르게 해석하기 때문에 결과를 신뢰할 수 없다.</p>
<hr />
<h2 id="올바른-포인터-사용">올바른 포인터 사용</h2>
<p><code>int</code> 변수에는 <code>int *</code>를 사용한다.</p>
<pre><code class="language-c">#include &lt;stdio.h&gt;

int main(void)
{
    int num = 10;
    int *ptr = &amp;num;

    printf(&quot;%d\n&quot;, *ptr);

    return 0;
}</code></pre>
<p><code>double</code> 변수에는 <code>double *</code>를 사용한다.</p>
<pre><code class="language-c">#include &lt;stdio.h&gt;

int main(void)
{
    double num = 3.14;
    double *ptr = &amp;num;

    printf(&quot;%f\n&quot;, *ptr);

    return 0;
}</code></pre>
<p>정리하면 다음과 같다.</p>
<pre><code class="language-text">int 변수       -&gt; int *
char 변수      -&gt; char *
double 변수    -&gt; double *
float 변수     -&gt; float *</code></pre>
<hr />
<h1 id="최종-정리">최종 정리</h1>
<h2 id="부호-비트-1">부호 비트</h2>
<ul>
<li>signed 자료형의 가장 왼쪽 비트이다.</li>
<li><code>1</code>이면 음수, <code>0</code>이면 양수이다.</li>
<li>음수는 보통 2의 보수 방식으로 표현된다.</li>
<li>signed 음수를 오른쪽 시프트하면 보통 빈 공간이 부호 비트로 채워진다.</li>
<li>unsigned 자료형은 오른쪽 시프트 시 빈 공간이 항상 <code>0</code>으로 채워진다.</li>
<li>signed 자료형의 시프트는 오버플로우나 정의되지 않은 동작을 만들 수 있으므로 주의해야 한다.</li>
</ul>
<h2 id="do-while">do while</h2>
<ul>
<li>조건식과 상관없이 최소 한 번은 실행된다.</li>
<li>실행 후 조건식을 검사한다.</li>
<li>조건식이 참이면 반복하고, 거짓이면 종료한다.</li>
</ul>
<h2 id="break-1">break</h2>
<ul>
<li>반복문이나 <code>switch</code>문을 빠져나갈 때 사용한다.</li>
</ul>
<h2 id="continue-1">continue</h2>
<ul>
<li><code>continue</code> 아래 코드를 건너뛰고 다음 반복으로 넘어간다.</li>
</ul>
<h2 id="포인터-1">포인터</h2>
<ul>
<li>변수는 메모리에 저장된다.</li>
<li><code>&amp;변수</code>는 변수의 주소를 의미한다.</li>
<li>포인터는 주소를 저장하는 변수이다.</li>
<li><code>int *ptr</code>은 <code>int</code>형 변수를 가리키는 포인터이다.</li>
</ul>
<h2 id="역참조-1">역참조</h2>
<ul>
<li><code>*ptr</code>은 <code>ptr</code>이 가리키는 주소에 직접 접근한다.</li>
<li>값을 가져올 수도 있고 저장할 수도 있다.</li>
<li><code>int *</code>를 역참조하면 <code>int</code>가 된다.</li>
<li>포인터의 자료형과 실제 변수의 자료형이 다르면 위험하다.</li>
</ul>
