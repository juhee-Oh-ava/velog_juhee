<h3 id="기본-타입과-참조-타입">기본 타입과 참조 타입</h3>
<blockquote>
<p>기본 타입인 byte, char, short, int, long, float, double, boolean 변수는 실제 &quot;값&quot;을 변수 안에 저장한다.</p>
</blockquote>
<blockquote>
<p>참조 타입인 배열, 열거, 클래스, 인터페이스 변수는 &quot;메모리의 번지&quot;를 변수 안에 저장한다.</p>
</blockquote>
<pre><code class="language-java">//기본 타입

int age = 25;
double price = 100.5;</code></pre>
<pre><code class="language-java">// 참조 타입 

String name = &quot;신용권&quot;;
String hobby = &quot;독서&quot;;</code></pre>
<p><img alt="" src="https://velog.velcdn.com/images/miy7625/post/5eb77e10-f326-4820-a24a-e2ced1742c2f/image.jpg" /></p>
<blockquote>
<p>메소드 영역 : JVM이 시작할 때 생성되고 모든 스레드가 공유하는 영역. 코드에서 사용되는 클래스들을 클래스 로더로 읽어 클래스별로 static field와 constant, 메소드 코드, 생성자 코드 등을 분류해서 저장한다. </p>
</blockquote>
<blockquote>
<p>힙 영역: 객체와 배열이 생성되는 영역. 생성된 객체와 배열은 JVM 스택 영역의 변수나 다른 객체의 필드에서 참조한다. </p>
</blockquote>
<blockquote>
<p>JVM 스택 영역: 메소드를 호출할 때마다 프레임을 추가하고 메소드가 종료되면 해당 프레임을 제거하는 동작을 수행</p>
</blockquote>
<h3 id="참조-변수의---연산">참조 변수의 ==, != 연산</h3>
<p>기본 타입 변수의 ==, != 연산은 변수의 값이 같은지, 아닌지를 조사하지만,
참조 타입 변수의 ==, != 연산은 동일한 객체를 참조하는지, 다른 객체를 참조하는지 알아볼 때 사용된다.</p>
<p>=&gt; 동일한 객체를 참조하고 있을 경우 == 연산의 결과는 true 이고 != 연산의 결과는 false 이다. </p>
<h3 id="null-과-nullpointerexception">null 과 NullPointerException</h3>
<p>참조 타입 변수는 힙 영역의 객체를 참조하지 않는다는 뜻으로 null 값을 가질 수 있다. null 값도 초기값으로 사용할 수 있어 null로 초기화된 참조 변수는 스택 영역에 생성된다.</p>
<blockquote>
<p>NullPointerException : 참조 타입 변수를 잘못 사용할때 발생</p>
</blockquote>
<ul>
<li>만약 null 상태에서 있지도 않은 객체의 데이터(필드)나 메소드를 사용하는 코드를 실행하면 NullPointerException 발생한다.</li>
</ul>
<pre><code>예시) 문자열은 String 객체로 생성되고 변수는 String 객체를 참조한다. 
name 변수와 hobby 변수는 스택 영역에 생성되고, 문자열 리터럴인 &quot;신용권&quot;과 &quot;자바&quot;는 힙 영역에 String 객체로 생성된다. 
그리고 name 변수와 hobby 변수에는 String 객체의 번지 값이 저장된다.</code></pre><blockquote>
<p>자바는 문자열 리터럴이 동일하다면 String 객체를 공유한다.</p>
</blockquote>
<pre><code class="language-java">String name1 = &quot;신용권&quot;;
String name2 = &quot;신용권&quot;;
</code></pre>
<blockquote>
<p>new 연산자를 사용해서 직접 String 객체를 생성시킬 수 있다. new 연산자는 힙 영역에 새로운 객체를 만들 때 사용하는 연산자로 <strong>객체 생성 연산자</strong> 라고 한다. </p>
</blockquote>
<pre><code class="language-java">String name1 = new String(&quot;신용권&quot;);
String name2 = new String(&quot;신용권&quot;);</code></pre>
<ul>
<li>동일한 String 객체이건 다른 String 객체이건 상관없이 내부 문자열을 비교하고 싶을 때는 String 객체의 <strong>equals()</strong> 메소드를 사용해야 한다. </li>
</ul>
<h2 id="배열">배열</h2>
<pre><code class="language-java">int sum = 0;
for(int i = 0; i &lt; 30; i++){
    sum += score[i];
}
int avg = sum / 30;</code></pre>
<blockquote>
<ol>
<li>배열은 같은 타입의 데이터만 저장할 수 있다.</li>
<li>한 번 생성된 배열은 길이를 늘리거나 줄일 수 없다. </li>
</ol>
</blockquote>
<ul>
<li>배열 변수는 참조 변수에 속하고, 배열도 객체이므로 힙 영역에 생성된다. 따라서 배열 변수는 힙 영역의 배열 객체를 참조하게 된다.</li>
</ul>

        }
        for(int i = 0; i &lt; newIntArray.length; i++){
            System.out.println(newIntArray[i] + &quot; ,&quot;);
        }
       }
    }</code></pre>
