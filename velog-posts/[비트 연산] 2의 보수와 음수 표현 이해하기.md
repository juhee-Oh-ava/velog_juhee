<h1 id="오늘-배운-내용-정리-비트-연산과-2의-보수">오늘 배운 내용 정리: 비트 연산과 2의 보수</h1>
<h2 id="1-부호-있는-정수와-2의-보수">1. 부호 있는 정수와 2의 보수</h2>
<p>컴퓨터에서 <strong>부호 있는 음수</strong>를 2진수로 표현할 때는 보통 <strong>2의 보수 방식</strong>을 사용한다.</p>
<p>부호 있는 8비트, 즉 1바이트 변수의 표현 범위는 다음과 같다.</p>
<pre><code class="language-text">-128 ~ 127</code></pre>
<p>8비트는 총 256개의 값을 표현할 수 있다.</p>
<p>부호 있는 정수에서는 이 256개의 표현을 음수와 양수에 나누어 사용한다.</p>
<pre><code class="language-text">음수: -128 ~ -1
0: 0
양수: 1 ~ 127</code></pre>
<p>따라서 <code>-145</code>는 8비트에 담을 수 없다.</p>
<pre><code class="language-text">8비트 signed 정수 범위: -128 ~ 127
-145는 이 범위를 벗어남</code></pre>
<p>그래서 <code>-145</code> 같은 값은 16비트 이상의 자료형이 필요하다.</p>
<hr />
<h2 id="2-음수를-비트로-표현하는-방법">2. 음수를 비트로 표현하는 방법</h2>
<p>음수가 있는 비트 계산은 일반적으로 아래 순서로 이해할 수 있다.</p>
<p>예를 들어 <code>-5</code>를 8비트로 표현한다고 해보자.</p>
<h3 id="1-절대값의-2진수를-구한다">1. 절대값의 2진수를 구한다</h3>
<pre><code class="language-text">5 = 0000 0101</code></pre>
<h3 id="2-1의-보수를-구한다">2. 1의 보수를 구한다</h3>
<p>1의 보수는 모든 비트를 반전시키는 것이다.</p>
<pre><code class="language-text">0000 0101
↓ 비트 반전
1111 1010</code></pre>
<h3 id="3-2의-보수를-구한다">3. 2의 보수를 구한다</h3>
<p>2의 보수는 1의 보수에 <code>1</code>을 더한 것이다.</p>
<pre><code class="language-text">1111 1010
+       1
---------
1111 1011</code></pre>
<p>따라서 8비트에서 <code>-5</code>는 다음과 같이 표현된다.</p>
<pre><code class="language-text">-5 = 1111 1011</code></pre>
<hr />
<h2 id="3-왜-2의-보수를-만들-때-1을-더할까">3. 왜 2의 보수를 만들 때 1을 더할까?</h2>
<p>처음에는 이 부분이 잘 이해되지 않았다.</p>
<blockquote>
<p>왜 1의 보수에서 끝내지 않고, 굳이 1을 더해서 2의 보수를 만들까?</p>
</blockquote>
<p>핵심 이유는 다음과 같다.</p>
<blockquote>
<p><strong>0을 두 개로 만들지 않기 위해서이다.</strong></p>
</blockquote>
<p>그리고 그 결과로 덧셈과 뺄셈을 하나의 회로로 깔끔하게 처리할 수 있다.</p>
<hr />
<h2 id="4-1을-더하지-않으면-생기는-문제-0과--0">4. 1을 더하지 않으면 생기는 문제: +0과 -0</h2>
<p>만약 1의 보수까지만 사용하고, 마지막에 <code>1</code>을 더하지 않는다고 해보자.</p>
<p>4비트 예시로 보면 다음과 같다.</p>
<h3 id="양수-0">양수 0</h3>
<pre><code class="language-text">+0 = 0000</code></pre>
<h3 id="음수-0">음수 0</h3>
<p>1의 보수 방식에서는 양수 0의 비트를 반전하면 음수 0처럼 보이는 값이 생긴다.</p>
<pre><code class="language-text">+0 = 0000
비트 반전
-0 = 1111</code></pre>
<p>즉, 0이 두 개가 된다.</p>
<pre><code class="language-text">0000 = +0
1111 = -0</code></pre>
<p>이것을 수직선으로 보면 다음과 같다.</p>
<pre><code class="language-text">1의 보수 방식에서의 수직선

...  -3   -2   -1   -0   +0   +1   +2   +3  ...
      |    |    |    |    |    |    |    |</code></pre>
<p>여기서 문제는 <code>0</code>이라는 값은 수학적으로 하나여야 하는데, 컴퓨터 내부 표현에서는 <code>+0</code>과 <code>-0</code>이 따로 생긴다는 점이다.</p>
<hr />
<h2 id="5-2의-보수에서는-0이-하나만-존재한다">5. 2의 보수에서는 0이 하나만 존재한다</h2>
<p>2의 보수 방식에서는 1의 보수에 <code>1</code>을 더한다.</p>
<p>4비트에서 <code>0</code>을 예로 들면 다음과 같다.</p>
<pre><code class="language-text">+0 = 0000</code></pre>
<p><code>-0</code>을 만들려고 해보자.</p>
<pre><code class="language-text">0000
↓ 비트 반전
1111
+   1
-----
0000</code></pre>
<p>4비트에서는 자리올림이 넘쳐서 버려지기 때문에 결과는 다시 <code>0000</code>이 된다.</p>
<p>즉, 2의 보수에서는 <code>+0</code>과 <code>-0</code>이 따로 존재하지 않는다.</p>
<pre><code class="language-text">0 = 0000</code></pre>
<p>수직선으로 보면 훨씬 자연스럽다.</p>
<pre><code class="language-text">2의 보수 방식에서의 수직선

...  -4   -3   -2   -1    0   +1   +2   +3  ...
      |    |    |    |    |    |    |    |</code></pre>
<p>0이 하나만 존재하기 때문에 표현 범위를 더 효율적으로 사용할 수 있다.</p>
<hr />
<h2 id="6-4비트-기준으로-비교하기">6. 4비트 기준으로 비교하기</h2>
<p>4비트 signed 정수를 기준으로 비교하면 차이가 더 잘 보인다.</p>
<h3 id="1의-보수-방식">1의 보수 방식</h3>
<pre><code class="language-text">0111 = +7
0110 = +6
0101 = +5
0100 = +4
0011 = +3
0010 = +2
0001 = +1
0000 = +0

1111 = -0
1110 = -1
1101 = -2
1100 = -3
1011 = -4
1010 = -5
1001 = -6
1000 = -7</code></pre>
<p>1의 보수 방식에서는 <code>+0</code>과 <code>-0</code>이 모두 존재한다.</p>
<p>그래서 4비트로 표현 가능한 값의 범위는 다음과 같다.</p>
<pre><code class="language-text">-7 ~ +7</code></pre>
<p>0이 두 개라서 하나의 표현을 낭비하게 된다.</p>
<hr />
<h3 id="2의-보수-방식">2의 보수 방식</h3>
<pre><code class="language-text">0111 = +7
0110 = +6
0101 = +5
0100 = +4
0011 = +3
0010 = +2
0001 = +1
0000 =  0

1111 = -1
1110 = -2
1101 = -3
1100 = -4
1011 = -5
1010 = -6
1001 = -7
1000 = -8</code></pre>
<p>2의 보수 방식에서는 0이 하나만 존재한다.</p>
<p>그래서 4비트로 표현 가능한 값의 범위는 다음과 같다.</p>
<pre><code class="language-text">-8 ~ +7</code></pre>
<p>즉, 2의 보수 방식은 <code>-0</code>을 없애고, 그 자리를 가장 작은 음수인 <code>-8</code> 표현에 사용한다.</p>
<hr />
<h2 id="7-2의-보수의-장점">7. 2의 보수의 장점</h2>
<p>2의 보수를 사용하는 이유는 단순히 음수를 표현하기 위해서만은 아니다.</p>
<p>가장 큰 장점은 다음과 같다.</p>
<blockquote>
<p><strong>덧셈 회로 하나로 덧셈과 뺄셈을 모두 처리할 수 있다.</strong></p>
</blockquote>
<p>예를 들어,</p>
<pre><code class="language-text">5 - 3</code></pre>
<p>은 다음처럼 바꿀 수 있다.</p>
<pre><code class="language-text">5 + (-3)</code></pre>
<p>컴퓨터 입장에서는 뺄셈 회로를 따로 복잡하게 만들 필요 없이, 음수를 2의 보수로 바꾼 뒤 덧셈으로 처리하면 된다.</p>
<hr />
<h2 id="8-일반적인-int-자료형은-왜-32비트일까">8. 일반적인 int 자료형은 왜 32비트일까?</h2>
<p>일반적인 <code>int</code> 자료형은 보통 32비트이다.</p>
<p>그 이유는 메모리 효율성과 표현 범위 사이의 균형이 좋기 때문이다.</p>
<p>32비트 signed int는 대략 다음 범위를 표현할 수 있다.</p>
<pre><code class="language-text">-2,147,483,648 ~ 2,147,483,647</code></pre>
<p>즉, 약 <code>-21억</code>부터 <code>+21억</code>까지 표현할 수 있다.</p>
<p>32비트는 대부분의 일반적인 정수 계산에 충분한 범위를 제공하면서도, 메모리를 너무 많이 쓰지 않는 적절한 크기이다.</p>
<hr />
<h2 id="9-signed와-unsigned">9. Signed와 Unsigned</h2>
<p>정수형에는 크게 두 가지 방식이 있다.</p>
<h2 id="signed">Signed</h2>
<p><code>Signed</code>는 부호가 있는 변수이다.</p>
<p>맨 앞 비트, 즉 최상위 비트를 부호 비트로 사용한다.</p>
<pre><code class="language-text">0이면 양수
1이면 음수</code></pre>
<p>예를 들어 8비트 signed 정수에서 다음 값은 최상위 비트가 <code>1</code>이므로 음수로 해석된다.</p>
<pre><code class="language-text">1111 1011</code></pre>
<p>이 값은 2의 보수 방식으로 해석하면 <code>-5</code>이다.</p>
<hr />
<h2 id="unsigned">Unsigned</h2>
<p><code>Unsigned</code>는 부호가 없는 변수이다.</p>
<p>음수라는 개념 자체가 없고, 0과 양수만 존재한다.</p>
<p>따라서 맨 앞 비트가 <code>1</code>이더라도 부호를 뜻하지 않는다.</p>
<p>그냥 숫자의 크기를 나타내는 비트로 사용된다.</p>
<p>예를 들어 8비트 unsigned 정수에서 다음 값은 음수가 아니다.</p>
<pre><code class="language-text">1111 1011</code></pre>
<p>이 값은 unsigned로 해석하면 다음과 같다.</p>
<pre><code class="language-text">251</code></pre>
<hr />
<h2 id="10-같은-비트라도-타입에-따라-해석이-달라진다">10. 같은 비트라도 타입에 따라 해석이 달라진다</h2>
<p>중요한 점은 비트 자체가 음수인지 양수인지를 결정하는 것이 아니라는 점이다.</p>
<blockquote>
<p><strong>같은 비트라도 자료형이 signed인지 unsigned인지에 따라 해석이 달라진다.</strong></p>
</blockquote>
<p>예를 들어 다음 비트를 보자.</p>
<pre><code class="language-text">1111 1011</code></pre>
<p>이 비트를 signed로 해석하면 다음과 같다.</p>
<pre><code class="language-text">-5</code></pre>
<p>하지만 unsigned로 해석하면 다음과 같다.</p>
<pre><code class="language-text">251</code></pre>
<p>즉, 비트 패턴은 같지만 자료형에 따라 의미가 달라진다.</p>
<hr />
<h2 id="11-자바와-cc의-차이">11. 자바와 C/C++의 차이</h2>
<p>자바의 기본 정수형들은 전부 signed이다.</p>
<p>예를 들어 자바의 <code>byte</code>, <code>short</code>, <code>int</code>, <code>long</code>은 모두 부호 있는 정수형이다.</p>
<p>따라서 자바에서는 최상위 비트가 <code>1</code>이면 음수로 해석된다.</p>
<p>반면 C/C++에서는 변수 타입에 따라 달라질 수 있다.</p>
<pre><code class="language-c">signed int a;
unsigned int b;</code></pre>
<p>C/C++에서는 <code>signed</code>인지 <code>unsigned</code>인지에 따라 같은 비트도 다르게 해석된다.</p>
<hr />
<h1 id="정리">정리</h1>
<p>오늘 배운 핵심은 다음과 같다.</p>
<ul>
<li>음수는 보통 2의 보수 방식으로 표현한다.</li>
<li>2의 보수는 1의 보수에 1을 더해서 만든다.</li>
<li>1을 더하는 이유는 <code>+0</code>과 <code>-0</code>이 따로 생기는 문제를 없애기 위해서이다.</li>
<li>2의 보수 덕분에 0은 하나만 존재한다.</li>
<li>2의 보수 방식에서는 덧셈과 뺄셈을 하나의 덧셈 회로로 처리할 수 있다.</li>
<li>8비트 signed 정수의 범위는 <code>-128 ~ 127</code>이다.</li>
<li>일반적인 <code>int</code>는 보통 32비트이며, 약 <code>-21억 ~ +21억</code>까지 표현할 수 있다.</li>
<li>signed는 음수를 표현할 수 있고, unsigned는 0과 양수만 표현한다.</li>
<li>같은 비트라도 signed인지 unsigned인지에 따라 값이 다르게 해석된다.</li>
</ul>
